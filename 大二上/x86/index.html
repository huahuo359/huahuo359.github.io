    1. 寻指方式和指令系统 ¶

    1.1 操作数寻址方式 ¶

    • 立即数方式: 操作数为常数
    • 寄存器方式: 利用寄存器进行操作
    • 直接寻址: 段地址 (ds/cs/es/ss) + 偏移地址 ( 立即数形式 )
    • 间接寻址: 基指寄存器 (bx/bp) + 变址寄存器 (si/di) + 位移量 ( 立即数 )

    其中 bx/bp 最多有其中一个 , si/di 最多有其中一个 .

    [bx+ 位移 ] 可以有几种变形 : [bx][ 位移 ], [bx]+[ 位移 ], [bx]+ 位移 , 位移 +[bx], 位移 [bx]

    [bx+si+ 位移 ] 的变形可以仿照上面的形式进行

    在使用 bp 时默认的缺省段地址是 ss , 其他情况下缺省的段地址都是 ds , 需要使用其他段地址时需要明确写出

    1.2 指令系统 ¶

    1.2.1 数据传送指令 ¶

    1.2.1.1 通用数据传送指令 ¶

    MOV

    • 指令格式 : mov dest, src
    • 指令作用 : 把 src 传给 dest

    • 注意 : dest可以是寄存器/mem, src可以是寄存器/mem/idata

      不可以两者都是 mem , 并且将立即数赋值给逻辑地址时要指明数据大小 ( word ptr/byte ptr/dword ptr ), 数据在内存中以小端形式进行存储

      立即数不可以赋值给段寄存器 , 段寄存器之间也不可以相互赋值 . 要借助普通寄存器 ( 如 ax) 进行赋值 .

      寄存器 CS 不能进行赋值 .

      MOV 指令无法访问寄存器 IP 与 FL

      赋值时的宽度必须一致 .

      MOV 指令不会对标志位产生影响 .

    PUSH

    • 指令格式 : push op 将 op 压入栈中
    • 指令作用 : sp=sp-2, word ptr ss:[sp] = op
    • 注意 : op 只能是寄存器或逻辑地址 , 不可以是立即数 , 不可以是 fl. op 的大小只能是 16 位 . 不影响标志位 .

    POP

    • 指令格式 : pop op ; 从栈中弹出一个 16 位的数据到 op 中
    • 指令作用 : op = word ptr ss:[sp], sp=sp+2
    • 注意 : op 是寄存器或者逻辑地址，一定是 16 位，不能是 cs fl。不影响任何标志位

    XCHG

    • 指令格式 : xchg op1, op2
    • 指令作用 : 交换 op1 和 op2 的值
    • 注意 : op1 op2 是寄存器或者内存，不能都是内存，且不能是段寄存器。不影响任何标志位
    1.2.1.2 端口输入输出指令 (in/out) ¶

    IN

    • 指令格式 : in acc, port
    • 指令作用 : 从端口读数据到 al 或 ax 中，即 acc = [port]（端口中内容）

    • 注意 : acc 一定是 al 或 ax，port 一定是 dx 或立即数，如果是立即数则最大 0FFh。是 ax 则 [port] 存入 al，[port+1] 存入 ah OUT

    • 指令格式 : out port, acc

    • 指令作用 : 把 al 或 ax 中值写入端口，即 [port] = acc
    • 注意 : port 一定是 dx 或立即数（立即数最大 0FFh），acc 一定是 al 或 ax。是 ax 则 al 写入 [port]，ah 写入 [port+1]
    1.2.1.3 地址传送指令 (lea/lds/les) ¶

    LEA

    • 指令格式 : lea dest, src ; 取变量的偏移地址 (Load Effective Address)
    • 指令作用 : dest = offset src
    • 注意 : dest 是寄存器，src 是内存变量。取 src 的偏移地址
    • 补充 : 利用 lea 指令可以进行四则运算 :

    lea dx, ds:[bx+si+3]: dx=bx+si+3 ; 如果直接使用 mov 指令则是语法错误

    lea eax, ds:[eax+eax*2] ; 利用 lea 做乘法

    LDS

    • 指令格式 : lds dest, src ; 将远指针装载到 ds:dest 中
    • 指令作用 : lds reg, mem mem 中是一个 double word 代表一个远指针 ( 段地址 + 偏移地址 ). 指令 将段地址赋值给ds, 偏移地址赋值给reg
    • 注意 : dest 是寄存器，src 是双字大小内存

    LES

    • 指令格式 : les dest, src
    • 指令作用 : 与 lds 类似 , 将 src 处远指针装入 es:dest，即 dest = word ptr [src], es = word ptr [src+2]
    • 注意 :dest 是寄存器，src 是双字大小内存
    1.2.1.4 标志寄存器传送指令 (LAHF,SAHF,PUSHF,POPF) ¶

    PUSHF/POPF

    • 指令格式 : pushf/popf
    • 指令作用 : 把标志寄存器 FL 压入堆栈 / 弹出堆栈
    • 注意 : 不能写成 push fl/pop fl, fl 与 ip 都不可以直接被使用

    stc clc: 修改 CF

    std cld: 修改 DF

    sti cli: 修改 IF

    可以通过 pushf 与 popf 刻意对某些位进行修改 , 如 :TF 不存在指令对其进行修改

    LAHF - 指令格式: lahf - 指令作用: 将fl的低8位复制到ah中

    SAHF - 指令格式: sahf - 指令作用: 将 ah 存入 fl 低 8 位，即 fl = (fl & 0FF00h) | 2 | (ah & 0D5h) - 注意: 只保留 ah 的 0 2 4 6 7 位（cf pf af zf sf），且第 1 位恒为 1

    1.2.2 转换指令 ¶

    CBW

    • 指令格式 :cbw ; 有符号拓展
    • 指令作用 : 将字节 al 扩充为字 ax，即将 al 符号位扩展到 ah

    CWD

    • 指令格式 : cwd
    • 指令作用 : 将字 ax 扩充为双字 dx:ax（直接拼接），即将 ax 符号位扩展到 dx

    XLAT

    • 指令格式 :xlat ; 查表指令
    • 指令作用 : 换码指令，al = byte ptr ds:[bx+al]
    • 注意 : 一般使 ds:bx 指向一张表，然后功能即类似列表索引取值 ( 用 al 进行下标的索引 , 结果存储在 al 中 )

    1.2.3 算术指令 ¶

    1.2.3.1 代数运算 ¶

    ADD

    • 指令格式 :add dest, src
    • 指令作用 : dest = dest + src
    • 注意 :src 可以是立即数 / 寄存器 / 内存，dest 可以是寄存器 / 内存，但二者不能都为内存，宽度要一致。可能会产生进位（cf）和溢出（of）
    • 补充 : CF : 无符号数的进位 . OF : 符号数的 overflow

    INC

    • 指令格式 : inc op
    • 作用 : 将 op 加一，即 op += 1
    • 注意 : op 为寄存器或内存。不影响 cf 位 , 会影响 zf 位 .

    ADC

    • 指令格式 : adc dest, src
    • 作用 : 带进位加，即 dest = dest + src + cf
    • 注意 : 操作数规则与 add 相同，可以用于模拟 32 位加法

    SUB

    • 指令格式 : sub dest, src
    • 指令作用 : 从 dest 减去 src，即 dest -= src
    • 注意 : 操作数格式同 add，会产生溢出（of），借位也会使 cf 变 1

    SBB

    • 指令格式 : sbb dest, src
    • 指令作用 : 带借位减，dest = dest - src - cf
    • 注意 : 操作数格式同 add

    DEC

    • 指令格式 : dec op ; 具体细节和 inc 相同

    NEG

    • 指令格式 : neg op
    • 作用 : op = -op. 求相反数 ( 有符号数表示 ) 即 : 取反 + 1
    • 注意 : op 为寄存器或内存 . op = 0 - op 非零数求补后 cf=1，0 求补后 cf=0

    CMP

    • 指令格式 : cmp op1, op2
    • 作用 : 用 op1 减 op2，但丢弃结果只影响标志位
    • 注意 : 操作数格式同 sub，用于在 jump 指令前给出符号位作为条件判断依据。
    • 补充 : 通常搭配指令 jxx dest 进行条件跳转 .

    MUL

    • 指令格式 : mul src ; 无符号乘法

    • 作用 : 8 位 : AX = AL * src

      16 位 : DX:AX = AX * src - 注意: src为reg/mem

    IMUL

    • 指令格式 : imul src ; 符号数乘法
    • 指令作用：符号数乘法，乘法方式同 mul，但将两个乘数和积都视为符号数

    DIV

    • 指令格式 : div op
    • 指令作用 : 无符号数除法
      • 当 op 为 8 位时，al = ax / op, ah = ax % op
      • 当 op 为 16 位时，ax = dx:ax / op, dx = dx:ax % op
    • 注意 : op 是寄存器或内存，可以得到商和余数

    IDIV - 指令格式: idiv op - 指令作用：符号数除法，除法操作同 div，但将除数、被除数、商和余数都视为符号数

    1.2.3.3 逻辑运算 ¶

    1. 逻辑运算指令 : AND, OR, XOR, NOT , TEST

    注意 : 用法与算数运算的方法类似 . TEST 进行的运算为 & 运算 , 作用与 cmp 类似 .

    2. 移位指令 :

    SHL

    • 指令格式 : shl dest, count ; 逻辑左移
    • 指令作用 : 对 dest 逻辑左移 count 位，右侧补 0，左侧最后溢出的一位落入 cf
    • 注意 : dest 可以是寄存器或内存，count 是 1 或 cl（8086 里不能为其它内容）

    SHR

    • 指令格式 : shr dest, count
    • 指令作用：对 dest 逻辑右移 count 位，左侧补 0，右侧最后溢出的一位落入 cf
    • 注意：dest 可以是寄存器或内存，count 是 1 或 cl（8086 里不能为其它内容）

    SAL

    • 指令格式 : sal dest, count
    • 指令作用 : 对 dest 算数左移 count 位，同逻辑左移
    • 注意：与 shl 完全相同

    SAR

    • 指令格式：sar dest, count
    • 指令作用：对 dest 算数右移 count 位，左侧补符号位，右侧最后溢出的一位落入 cf
    • 注意：dest 可以是寄存器或内存，count 是 1 或 cl（8086 里不能为其它内容）

    ROL

    • 指令格式：rol dest, count
    • 指令作用：对 dest 循环左移 count 位，最高位回到最低位同时移到 cf 中
    • 注意：dest 可以是寄存器或内存，count 是 1 或 cl（8086 里不能为其它内容）。

    ROR

    • 指令格式：ror dest, count
    • 指令作用：对 dest 循环右移 count 位，最低位回到最高位同时移到 cf 中
    • 注意：dest 可以是寄存器或内存，count 是 1 或 cl（8086 里不能为其它内容）。

    RCL

    • 指令格式：rcl dest, count
    • 指令作用： 带进位循环左移，即 cf 加在 dest 左侧一起循环左移
    • 注意：dest 可以是寄存器或内存，count 是 1 或 cl（8086 里不能为其它内容）。

    RCR

    • 指令格式：rcr dest, count
    • 指令作用：带进位循环右移，即 cf 加在 dest 右侧一起循环右移
    • 注意：dest 可以是寄存器或内存，count 是 1 或 cl（8086 里不能为其它内容）。

    1.2.4 十进制调整指令 ¶

    1.2.5 字符串操作指令 ¶

    1. 字符串传送指令 :MOVSB,MOVSW,MOVSD

    • 指令格式 : rep movsb
    • 指令作用 : 字符串拷贝。源字符串 : ds:si 、目标字符串：es:di. 拷贝次数 :cx 进行设置 . 由源字符串向目标字符串进行复制 .
    • 注意 : movsb: 每次复制一个 byte. movsw: 每次复制一个 word. movsd: 每次复制一个 dword.

    当 DF=0 时 , 进行正向复制 (cld). 当 DF=1, 进行反向复制 (std).

    2. 字符串比较指令：CMPSB，CMPSW，CMPSD

    • 指令格式 : cmpsb / repe cmpsb / repne cmpsb
    • 指令作用：比较字节 ds:[si] 与 es:[di]，即 byte ptr ds:[si] - byte ptr es:[di] 丢弃结果保留标志位，并移动 si di
    • 注意：si di 的移动不会影响标志位。si di 的移动与 df 有关，预先用 cld std 设置，df=0 则 si di 移向下一个字节，否则移向上一个。
    • eg: 对于 repe cmpsb( 如果本次比较相等则继续比较下一个 ), 比较后先移动 si 与 di, 如果不相等则结束 . 即 : 结束后会指向第一次出现不等的下一位
    • 当字符串全等时 :cx=0, 可以根据 ZF 进行判断 ,ZF=1 则双方全等

    3. 字符串扫描指令：scasb，scasw, scasd

    • 指令格式：scasb / repe scasb / repne scasb
    • 指令作用：比较 al 与 es:[di]，即计算 al - es:[di] 丢弃结果保留符号位，并移动 di
    • 注意：di 的移动不会影响标志位。di 的移动与 df 有关，预先用 cld std 设置，df=0 则 di 移向下一个字节，否则移向上一个

    4. 存入字符串指令 :STOSB STOSW STOSD

    • 指令格式：stosb / rep stosb
    • 指令作用：把字节 al 存入 es:[di] 中，并移动 di
    • 注意：di 的移动与 df 有关，预先用 cld std 设置，df=0 则 di 移向下一个字节，否则移向上一个

    5. 字符串存取指令 : lodsb/stosb lodsw/stosw lodsd/stosd

    STOSB

    • 指令格式：stosb / rep stosb
    • 指令作用：把字节 al 存入 es:[di] 中，并移动 di
    • 注意：di 的移动与 df 有关，预先用 cld std 设置，df=0 则 di 移向下一个字节，否则移向上一个

    LODSB

    • 指令格式 : lodsb ; 没有 rep 形式
    • 指令作用：从 ds:[si] 读取一个字节存入 al，并移动 si
    • 注意：si 的移动与 df 有关，预先用 cld std 设置，df=0 则 si 移向下一个字节，否则移向上一个 .

    通常将 lodsb 与 stosb 搭配进行使用 .

    编程时先设置 es 为 0B800h，然后计算出 bx = (y 80+x) 2，然后令 byte ptr es:[bx] = 字符，byte ptr es:[bx+1] = 颜色

    8.ss:sp 及堆栈段 :

    sp:stack pointer 堆栈指针 , 表示堆栈顶端的偏移地址 , 而ss则用来表示堆栈的段地址。

    但是不能用 [sp] 或 [sp+ 常数或其它寄存器 ] 的形式来引用某个变量。为了弥补 sp 不能表示间接地址的缺陷，同时也为了凑齐 4 个偏移地址寄存器，于是从通用寄存器中找来 bp 作为 sp 的替补 .

    在子程序设计时的堆栈框架上会经常使用 bp.

    9. 通过逻辑内存进行索引时可以使用 bx/bp + si/di + 常数 . 不可以使用其他的寄存器 .

    10.FL 标志寄存器 :

    FL 共 16 位 , 但只用其中 9 位，这 9 位包括 6 个状态标志和 3 个控制标志，如下所示：

    11 10 9 8 7 6 4 2 0

    D I T S Z A P C

    注意 : mov 指令不会对标志位造成影响

    CF: (carry flag) 借位 / 进位 搭配指令 : jc / jnc

    ZF: (zero flag) 搭配指令 : jz/jnz je/jne

    SF: (sign flag) 当符号位（最高位）为 1 时 SF = 1 。 搭配指令 : js/jns

    OF: (overflow flag) 符号数的溢出。 搭配指令 : jo/jno

    PF: 奇偶校验位 当有偶数个 1 时 PF=1. 并且 PF 只统计数据的低 8 位。 搭配指令 : jp/jnp

    AF: 辅助进位标志 当低 4 位向高 4 位产生进位 / 借位时 AF=1. 与 BCD 码相关。

    CF ZF SF OF AF PF: 这 6 个称为状态标志

    DF TF IF: 这 3 个称为控制标志

    DF:direction flag

    1
2
3
4
5
    cld 指令使DF=0, 进行正向操作

std 指令使DF=1, 进行反向操作

在进行字符串复制时应当注意采取正向还是逆向.

    TF:trace/trap flag

    1
    TF = 1 会进入单步执行的模式(调试)

    IF:interrupt flag

    1
2
3
    sti 指令使得IF=1, 即 允许中断

cli 指令使得IF=0, 即 禁止中断

    ; 运用rol指令把16位整数转化成16进制格式输出:
; 通过循环左移,每次取出4位进行判断,通过栈进行数据的保存
data segment
abc dw 32767
s db 4 dup(0),0Dh,0Ah,'$'
;4 dup(0)相当于0,0,0,0
;s[0]='7'; s[1]='F'; s[2]='F'; s[3]='F'
data ends
code segment
assume cs:code, ds:data
main:
   mov ax, data
   mov ds, ax
   mov ax, abc
   mov cx, 4
   mov di, 0; 目标数组的下标,可以引用s[di]
again:
   push cx
   mov cl, 4 ; 设ax的原值=7A9Dh
   rol ax, cl; AX=A9D7, 9D7A, D7A9, 7A9D
   push ax
   and ax, 0000000000001111B; 000Fh
   cmp ax, 10
   jb is_digit
is_alpha:
   sub al, 10
   add al, 'A'
   jmp finish_4bits
is_digit:
   add al, '0'
finish_4bits:
   mov s[di], al
   pop ax
   pop cx
   add di, 1
   sub cx, 1
   jnz again
   mov ah, 9
   mov dx, offset s
   int 21h
   mov ah, 4Ch
   int 21h
code ends
end main

input_done:
   mov a[bx], 0; 填充字符串结束标志
   mov ah, 2
   mov dl, 0Dh

   mov dl, 0Ah
   int 21h; 输出换行

   mov si, 0
output_next:
   mov dl, [bx+si]
   cmp dl, 0
   je output_done
   mov ah, 2
   int 21h; 输出DL中的字符
   add si, 1
   jmp output_next
output_done:
   mov ah, 4Ch

code ends
